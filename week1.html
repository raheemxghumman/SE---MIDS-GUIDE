Lecture 1: Introduction to Software Engineering

What is Software?

Software is more than just code. It is a complete system consisting of three main components that work together to deliver functionality.

Definition: Software

Software includes the program (code), data (information processed), and documentation (manuals, specs, API references). It is not just the executable file.

Program Data Documentation
  • Program — The actual code written in languages like Java, C++, Python, etc.
  • Data — The information the program processes, such as user profiles, images, or database records.
  • Documentation — All documents related to the software including user manuals, design documents, and API references.
📌 Example

Banking Application

Program: The code that handles login, balance check, and fund transfer.

Data: Your account number, balance, and transaction history.

Documentation: User guide, developer documentation, and installation guide.

The three components of software (Program, Data, Documentation) is a common MCQ question. Remember that software is NOT just the program.

The Nature of Software

Software has a dual nature. It can be viewed both as a product itself and as a vehicle for delivering other products.

Software as a Product

Delivers computing potential to users.

Produces, manages, acquires, modifies, displays, or transmits information.

Example: Microsoft Word is a product that helps create and edit documents.

Software as a Vehicle

Supports other systems such as operating systems, networks, and tools.

Provides the platform for other software to run.

Example: Windows OS is a vehicle that runs other software like games or browsers.

Software Industry Today
Modern software development involves teams of specialists working on complex systems. This is a shift from the lone programmer model of earlier times.

Persistent Questions in Software Development

Despite decades of progress in software engineering, certain fundamental questions remain challenging today just as they were in the past.

QuestionWhy It Matters
Why does it take so long to finish software?Complexity and changing requirements
Why are development costs so high?Need for skilled labor, tools, and extensive testing
Why can't we find all errors before delivery?Some bugs appear only in real-world use
Why so much time spent on maintenance?Software evolves with user needs
Why is progress hard to measure?Lack of physical tangibility
Important Insight
Maintenance often costs more than initial development. Fixing a bug after release can cost 100x more than fixing it during the design phase.

Same for everyone, cheaper, faster to get.

The difference between Generic (COTS) and Customized (Bespoke) products is frequently asked. Focus on who owns the specification.

Changing Nature of Software — Four Evolving Categories

a) WebApps

WebApps have evolved from static web pages with simple hyperlinks, images, and text into complex applications with interactivity, databases, and sophisticated business logic.

📌 Example

Google Docs

Started as a simple online document editor. Now offers real-time collaboration, cloud storage, version history, commenting, and integration with other services.

b) Mobile Applications

Mobile apps are designed specifically for mobile platforms such as iOS, Android, and Windows Mobile. They feature interoperability with web services and local processing capabilities.

Mobile WebApp

  • Accessed via browser
  • No installation needed
  • Works across platforms
  • Limited hardware access

Mobile App

  • Installed on device
  • Takes storage space
  • Platform-specific
  • Full hardware access (camera, GPS)

c) Cloud Computing

Definition: Cloud Computing

Cloud computing is an infrastructure or ecosystem that enables users to share computing resources from anywhere, providing scalable and on-demand services.

📌 Example

Cloud Platforms

AWS (Amazon Web Services), Google Cloud, and Microsoft Azure provide infrastructure, platforms, and software as services over the internet.

d) Product Line Software

Product line software refers to a set of software systems that share a common set of features for a specific market segment. These systems are developed from reusable core assets.

📌 Example

Android OS

Android runs on different phone models. All share core Android features, but manufacturers like Samsung, OnePlus, and Google Pixel customize the OS with their own skins and features.

Be able to contrast professional vs amateur development systematically. Focus on the process, team structure, and quality aspects.

Attributes of Good Software

Good software must satisfy four key quality characteristics to be considered professional and reliable.

Maintainability Dependability & Security Efficiency Acceptability
  1. Maintainability — The software should be easy to change and adapt to new requirements.
  2. Dependability & Security — The software must be reliable, safe, and secure against threats.
  3. Efficiency — The software should not waste system resources like memory or CPU time.
  4. Acceptability — Users should like the software and be able to use it effectively.

Definition of Software Engineering

Software engineering has been defined in multiple ways by different organizations and experts. All definitions emphasize systematic approaches and engineering discipline.

General Definition

An engineering discipline concerned with all aspects of software production, from initial specification to maintenance after deployment.

IEEE Definition

Application of a systematic, disciplined, quantifiable approach to the development, operation, and maintenance of software.

Another View

Establishment of sound engineering principles to obtain economical, reliable, and efficient software on real machines.

Key Points

Engineering discipline: Uses theories, methods, and tools to solve problems within organizational and financial constraints.

All aspects: Not just coding, but also project management, tool development, and process management.

Computer Science vs Software Engineering

Computer ScienceSoftware Engineering
Focuses on theory, algorithms, computationFocuses on practical production of software
Asks: Can we solve this?Asks: Can we build this on time and budget?
Example: New sorting algorithmExample: Building an e-commerce website
Relationship

Software Engineering is the process of utilizing our knowledge of computer science in the effective production of software systems.

System Engineering vs Software Engineering

System Engineering

Deals with the entire system including hardware, software, people, and processes.

Example: Designing a car (engine, brakes, software, assembly line).

Software Engineering

Focuses only on the software part of the system.

Example: Designing only the car's infotainment software.

Importance of Software Engineering

  • Society relies on software, so it must be reliable and trustworthy.
  • Using SE methods is cheaper in the long run compared to ad-hoc development.
  • The majority of costs come from post-delivery changes and maintenance.
Cost Reality
Fixing a bug after release costs 100x more than fixing it during the design phase. This is why systematic SE processes are essential.

Software Development Loop

Software development follows a simplified cyclical process:

Problem Definition → Technical Development → Solution Integration → Status Quo (then loop back)

A Software Engineering Framework (Layered Model)

The Layered Framework

Quality Focus (top layer) → Process → Methods → Tools (bottom layer)

  1. Quality Focus — Organizational commitment to continuous improvement and quality standards.
  2. Process Layer — Foundation that defines the framework, including products, milestones, quality assurance, and change management.
  3. Methods — Technical "how-to's" for communication, analysis, design, coding, and testing.
  4. Tools (CASE) — Computer-Aided Software Engineering tools that provide automated support for process and methods.
📌 Example

Framework in Practice

Tools: Git (version control)

Methods: Code reviews, design patterns

Process: Agile methodology

Quality Focus: ISO standards compliance

Memorize the order of the layered framework: Quality → Process → Methods → Tools. This appears frequently in exams.

Software Process Issues (Challenges)

a) Heterogeneity

Systems must run on different devices and networks, and often integrate with legacy systems.

📌 Example

Cross-Platform Challenge

An app running on iOS, Android, and Windows, while communicating with a 20-year-old mainframe system.

b) Business and Social Change

Markets demand faster delivery, but traditional development methods are slow.

c) Security and Trust

Software must prevent attacks and maintain information security.

📌 Example

Banking Security

A banking website must protect against hackers, implement encryption, and prevent unauthorized access to customer data.

d) Software Engineering Diversity

Different types of systems need different development approaches. There is no one-size-fits-all solution.

Software Engineering Fundamentals (Universal Principles)

These principles apply to all types of software systems, regardless of development methodology.

  1. Manage specification and requirements carefully.
  2. Deliver software within budget and schedule.
  3. Have a managed and understood development process.
  4. Ensure dependability and performance.
  5. Be effective with resources by reusing software components.

Software Engineering Practices (Four Steps)

  1. Understand the Problem — Who are stakeholders? What are the unknowns? Can the problem be compartmentalized? Can we create an analysis model?
  2. Plan a Solution — Have we seen similar problems before? Are there reusable elements? Can we create a design model?
  3. Carry Out the Plan — Does the code follow the design? Is it traceable? Has it been reviewed or proven correct?
  4. Examine the Result — Can we test each part? Do we have a reasonable testing strategy? Has it been validated against requirements?

Stakeholders in Software Engineering

Definition: Stakeholder

Anyone with a stake in the project's success. This includes users, customers, developers, and managers.

  • Users — People who actually use the software.
  • Customers — People who pay for or request the software.
  • Software developers — People who build the software.
  • Development managers — People who oversee the development process.
📌 Example

Hospital System Stakeholders

Users: Doctors, nurses

Customers: Hospital administration

Developers: Programmers

Managers: Project leads

Always include users, customers, developers, and managers when asked about stakeholders.
